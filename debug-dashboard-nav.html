Debug : Pourquoi la navigation va vers /login

Problèmes possibles :

  1. Route protection : Les routes `/dashboard/company` et `/candidate-dashboard` peuvent avoir des ProtectedRoute qui redirigent vers login
  2. Redirection automatique : Quelque part dans le code, il y a peut-être une redirection automatique vers login
  3. État utilisateur perdu : L'état d'authentification peut être perdu lors de la navigation
  4. Hook useAdminSystem : Peut-être qu'il vérifie les permissions et redirige

Solutions :

  1. Vérifier s'il y a des ProtectedRoute dans App.tsx
  2. Regarder si CompanyDashboard ou CandidateDashboard ont des useEffect qui redirigent
  3. Ajouter des console.log pour tracer l'état utilisateur

Test :

  1. Connectez-vous comme entreprise
  2. Allez sur le dashboard général (/dashboard)
  3. Cliquez "Mon Profil" → Accéder
  4. Regardez la console pour voir les logs de navigation
